Gitとの日常

作業用のブランチ(トピックブランチ)を作成 

# 参照するブランチを開発ブランチに切り替え
$ git checkout development 

# 開発ブランチを最新にする
$ git pull --rebase

# カレント(現在参照している)ブランチから、新しいブランチ(lovely-branch)を作成
$ git checkout -b lovely-branch

派生元のブランチに要注意!

  • 最新の開発(共有)ブランチから派生させるようにしましょう

何らかの作業

ローカルリポジトリに反映 

$ git status
$ git diff
$ git add hoge.php
$ git commit -m "add much more love to you, Git"
  • まとまった意味のあるcommit(="こういうことをしました"にまとめられるもの)をしましょう
  • リモートリポジトリとのやりとりは、基本的にはpushとpullだけ

リモートリポジトリに反映 & merge 

$ git push origin lovely-branch
  • github.comにアクセス
  • 開発ブランチにpull reqestを送る
  • merge

ちょっとした気遣い

.gitconfigを使う

# だいぶ省略

$ cat ~/.gitconfig 
[color]
        ui = auto
[alias]
        co = checkout
        ci = commit
        st = status
        br = branch
        lg = log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --date=relative
        lga = log --graph --all --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit --date=relative
[push]
        default = current
          ; git push したときにgit push origin {current_branch}の挙動になる 
[pull]
        rebase = true
          ; git pull したときにgit pull --rebaseしてくれる => 不必要なmerge commitが発生しなくなる

知ってるつもり - 誤解が生む悲劇

  • Gitは分散型リポジトリ!!!
  • Gitはコミットオブジェクトの集まり!!!
  • Gitにはたくさんのエイリアスがある!!!
ここが分かればGitの操作を直感的にできるようになれる

分散型リポジトリだから

  • どこに存在するリポジトリも、全部同じ!

    • bareリポジトリにはワーキングツリーがないとか、ホントは違うところがあるけど、リポジトリ管理者以外は知らなくとも大丈夫

  • 「ここをリモートリポジトリにしようぜ！」って言ったリポジトリがリモートリポジトリになる

    • cloneしてきた元のサーバのこと(originとも言う)。 
    $ git remote -v
origin  git@github.com:user_name/repo_name.git (fetch)
origin  git@github.com:user_name/repo_name.git (push)

    見方: gitプロトコルで@github.comというドメインにアクセスし:user_nameの/repo_name.gitからfetchする
